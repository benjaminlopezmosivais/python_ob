## Decoradores
un decorador es una funciona que modifica otra sin cambiar su codigo se usa con @

aqui declaro me funcion
def mi_decorador(func):
    def wrapper():
        print("Antes de ejecutar")
        func()
        print("Después de ejecutar")
    return wrapper

aqui la mando llamar y la peronalizo 
@mi_decorador
def saludar():

saludar()

# 1️⃣ Sí: el decorador recibe una función

Correcto:

def mi_decorador(func):

Aquí:

func = función que estás decorando

Ejemplo:

@mi_decorador  
def saludar():

Entonces:

func = saludar

---

# 2️⃣ El nombre del decorador puede ser cualquiera

Correcto. Puede llamarse como sea:

def decorador(func):  
def log(func):  
def medir_tiempo(func):  
def seguridad(func):

El nombre **no importa**, lo importante es que reciba una función.

---

# 3️⃣ El wrapper también puede llamarse como sea

Normalmente se llama `wrapper`, pero **no es obligatorio**.

Esto funciona igual:

def mi_decorador(func):  
  
    def cosa_rara():  
        func()  
  
    return cosa_rara

Se usa `wrapper` solo por **convención**.

---

# 4️⃣ Algo importante: el orden del código

Aquí te faltó un detalle.

Dijiste algo así:

func()  
print("Estoy entrando al wrapper")  
print("Estoy saliendo del wrapper")

Pero normalmente se hace así:

print("Antes")  
func()  
print("Después")

Ejemplo correcto:

def mi_decorador(func):  
  
    def wrapper():  
        print("Entrando")  
        func()  
        print("Saliendo")  
  
    return wrapper

---

# 5️⃣ Lo que hace `return wrapper`

Esto es **lo más importante del decorador**.

return wrapper

Significa:

la función original será reemplazada por wrapper

O sea:

Antes:

saludar → función original

Después del decorador:

saludar → wrapper

---

# 6️⃣ Qué hace el `@`

Tu intuición es correcta.

Esto:

@mi_decorador  
def saludar():

es exactamente lo mismo que escribir:

def saludar():  
    print("Hola")  
  
saludar = mi_decorador(saludar)

O sea:

la función se manda al decorador

# 7️⃣ Flujo completo

Código:

def mi_decorador(func):  
  
    def wrapper():  
        print("Antes")  
        func()  
        print("Después")  
  
    return wrapper  
  
  
@mi_decorador  
def saludar():  
    print("Hola")  
  
  
saludar()

Flujo real:

1 Python crea saludar  
2 Python hace mi_decorador(saludar)  
3 func = saludar  
4 se devuelve wrapper  
5 saludar = wrapper

Luego cuando ejecutas:

saludar()

realmente pasa:

wrapper()  
↓  
print("Antes")  
func()  → saludar original  
print("Después")

Salida:

Antes  
Hola  
Después

# 8️⃣ Tu explicación corregida (casi perfecta)

Lo que dijiste, corregido un poco, sería:

> Un decorador recibe una función (`func`).  
> Dentro tiene otra función (normalmente llamada `wrapper`) que ejecuta la función original y puede agregar código antes o después.  
> Luego se retorna `wrapper`, lo que hace que la función original sea reemplazada.  
> Usando `@decorador` Python automáticamente pasa la función al decorador.

Eso **ya es entender decoradores**. 🔥
